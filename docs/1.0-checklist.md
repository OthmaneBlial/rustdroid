# 1.0 Checklist

RustDroid is ready for `v1.0.0` only if every section below has an explicit yes.

## Testing

- unit tests cover core parsing, config, reporting, archive handling, and watch logic
- integration coverage exists for fixture flows and host runtime flows
- the smoke matrix is still the minimum critical loop
- failure artifacts remain readable after bad runs

## CI Stability

- fast checks are green on normal pushes
- host integration remains actionable instead of flaky noise
- package validation remains green
- performance baseline checks still catch large regressions

## Release Quality

- release notes are curated or intentionally generated
- install assets are verified before shipping
- rollback steps stay current
- the release process remains tag-driven and reproducible

## Install Quality

- `install.sh` still works from release assets and from source fallback
- uninstall behavior is predictable
- health checks still point users to real fixes

## Docs Quality

- install guide is current
- host backend guide is current
- troubleshooting guide reflects actual failures seen in CI and local use
- contribution and release docs remain usable without private context

## Performance Confidence

- host baseline thresholds remain realistic
- the fast loop still prefers host backend plus `scrcpy` or headless usage
- new features do not materially slow the normal APK loop

## Decision

Do not cut `v1.0.0` out of momentum alone.

Ship it only when the answer is explicit, not implied:

- yes, this is a mature public tool
- or no, more work is still required
