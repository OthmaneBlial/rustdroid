# RustDroid Release Announcement Checklist

- Confirm the GitHub release is published and assets are attached.
- Copy the install snippet from the release body.
- Summarize the top 3 release highlights in plain language.
- Include known limitations so users understand the current scope.
- Link the README and the release page.
- Mention the recommended first commands: `rustdroid doctor` and `rustdroid self-test`.
- Post the announcement only after the release artifacts download cleanly.
